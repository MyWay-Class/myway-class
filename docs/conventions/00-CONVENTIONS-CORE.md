# 00. 핵심 규칙

> 모든 레이어에 공통으로 적용되는 핵심 규칙

---

## 기본 원칙

- 코드는 읽기 쉬워야 한다.
- 요청받은 범위를 넘지 않는다.
- 문서와 구현은 서로 맞아야 한다.
- 추측성 기능은 추가하지 않는다.
- 변경은 작은 단위로 한다.
- 애플리케이션 코드는 TypeScript만 사용한다.
- JavaScript는 앱 코드에서 사용하지 않는다.
- JS가 꼭 필요한 경우는 도구 설정이나 빌드 스크립트처럼 경계가 분명한 곳으로 제한하고 이유를 문서에 남긴다.

## TypeScript 기본

- 타입 추론을 먼저 쓴다.
- `any`는 쓰지 않는다.
- 외부 입력은 `unknown`으로 받고 경계에서 좁힌다.
- 객체 계약은 `interface`, 조합 타입은 `type`을 우선한다.
- 공개 계약은 명확하게 export 한다.
- 기본형은 `string`, `number`, `boolean`, `bigint`, `symbol`, `null`, `undefined`를 우선한다.
- 고정 길이의 이종 데이터만 tuple로 표현한다.
- 값의 상태가 여러 개면 union type과 narrowing을 우선한다.
- `strictNullChecks`를 전제로 `null`과 `undefined` 가능성을 명시한다.
- 객체 타입은 `interface` 또는 `type`으로 표현하고, 경계에서만 `readonly`를 해제한다.

## 문서 규칙

- 한 문서는 한 책임만 가진다.
- 문서가 길어지면 하위 문서로 분리한다.
- AI 작업 규칙은 `docs/ai-context/`에 둔다.

## 공식 문서 기준

- TypeScript 공식 문서의 기본형, 객체 타입, narrowing 규칙을 따른다.
- JavaScript 공식 문서는 언어 자체와 표준 동작 확인용으로만 참고하고, 앱 코드 스타일 기준으로는 사용하지 않는다.
- 이 프로젝트의 코드 스타일 기준은 TypeScript 중심이며, JavaScript 전용 컨벤션은 별도로 두지 않는다.

## AI 작업 규칙

- 먼저 생각하고, 그 다음 수정한다.
- 관련 없는 파일은 건드리지 않는다.
- `apply_patch`를 우선한다.
- 결과는 검증 가능한 상태로 남긴다.
