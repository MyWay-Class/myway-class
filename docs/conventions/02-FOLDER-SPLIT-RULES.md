# 폴더 분리 규칙

## 목적
모노레포에서 폴더를 어떤 기준으로 나눌지 정한다.

## 기준
- 기능 책임이 다르면 폴더를 분리한다.
- 실행 환경이 다르면 폴더를 분리한다.
- 소유 주체가 다르면 폴더를 분리한다.
- 공통으로 재사용되는 것은 `shared`로 모은다.

## 현재 기본 구조
- `frontend/`: 사용자 화면
- `backend/`: API와 Workers
- `packages/shared/`: 공통 타입과 DTO
- `docs/`: 공통 설계와 운영 문서

## 예시
- 화면 로직과 API 호출은 `frontend/`로 보낸다.
- DB 접근과 AI 오케스트레이션은 `backend/`로 보낸다.
- 프론트와 백엔드가 같이 쓰는 타입은 `packages/shared/`로 보낸다.
- 프로젝트 공통 문서는 `docs/`에 둔다.

## 금지
- 프론트와 백엔드 코드를 같은 폴더에 섞지 않는다.
- `shared`에 UI 코드나 DB 코드를 넣지 않는다.
- `docs/`에 실행 코드나 테스트 코드를 넣지 않는다.

## 검증 기준
- 폴더만 봐도 책임이 구분된다.
- 새 기능을 어디에 둘지 애매하지 않다.
