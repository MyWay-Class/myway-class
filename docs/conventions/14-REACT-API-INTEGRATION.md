# React API 연동 규칙

## 목적
프론트에서 백엔드 API를 일관되게 호출한다.

## 규칙
- fetch 로직은 `frontend/src/lib/api.ts` 한곳에 모은다.
- 응답 형태는 공통 봉투를 따른다.
- API 타입은 `packages/shared`를 따른다.
- 오류는 화면에서 바로 이해할 수 있게 처리한다.
- 컴포넌트 안에 직접 `fetch`를 두지 않는다.

## 검증 기준
- 모든 API 호출이 공통 래퍼를 경유한다.
- 401과 5xx 처리가 화면에서 구분된다.
- 응답 타입이 공유 계약과 어긋나지 않는다.
