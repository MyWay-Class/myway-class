# 워킹트리 규칙

## 목적
작업 중인 파일 상태를 깨끗하게 유지한다.

## 규칙
- 작업 시작 전 현재 상태를 확인한다.
- 변경은 필요한 파일에만 만든다.
- 사용하지 않는 임시 파일은 남기지 않는다.
- 작업이 끝나면 문서와 코드 상태를 같이 확인한다.

## 권장 확인
- `git status`
- 변경된 파일 목록
- 문서와 코드의 일치 여부

## 다답안 비교
- 같은 이슈에 대해 2개 정도 접근을 비교한다.
- 변경 파일 수, 검증 결과, 변경량을 함께 본다.
- 채택 이유는 `docs/dev-logs/`에 짧게 남긴다.

## 금지
- 관계없는 파일을 몰아서 바꾸는 것
- 임시 산출물을 방치하는 것
- 작업이 끝났는데도 변경 범위를 설명하지 못하는 것

## 검증 기준
- 워킹트리를 봤을 때 무엇이 바뀌었는지 바로 알 수 있다.
- 불필요한 파일이 남아 있지 않다.
