# MoSCoW 우선순위 원칙

## 목적
기능과 문서의 우선순위를 같은 기준으로 정리한다.

## 정의
- `Must`: 반드시 있어야 하는 것
- `Should`: 있으면 매우 좋은 것
- `Could`: 있으면 좋지만 없어도 되는 것
- `Won't`: 이번 범위에서는 하지 않는 것

## 사용 방식
- 새 기능을 넣을 때 먼저 MoSCoW로 분류한다.
- 문서, 코드, 데모 범위를 정할 때도 같은 기준을 쓴다.
- `Must`가 아니면 이번 작업 범위에 넣지 않는 것을 기본 원칙으로 한다.

## 예시
- 강의 요약: `Must`
- 숏폼 공유: `Should`
- 랭킹 장식 효과: `Could`
- 고급 추천 알고리즘: `Won't`

## 검증 기준
- 우선순위를 말로 설명하지 않아도 분류가 바로 된다.
- 작업 범위를 줄이는 데 실제로 도움이 된다.
