# 개발 로그

> 작업 중 확인한 내용, 실험 결과, 이슈 메모를 적는 공간

## 문서 유형
기록 허브입니다. 결정 이유와 변경 흐름을 짧게 남깁니다.
로그는 원칙을 새로 만들기보다, 이미 정해진 기준을 어떻게 적용했는지 남기는 용도입니다.

---

## 사용 원칙
- 긴 설명은 본문 문서가 아니라 로그에 둔다.
- 임시 판단은 로그에 남기고, 확정 내용만 본문 문서에 반영한다.
- 날짜가 있는 기록은 변경 흐름 추적에만 사용한다.
- PR마다 변경 요약을 이 폴더에 남긴다.
- 로그는 `왜 바꿨는지`, `무엇을 바꿨는지`, `어떻게 검증했는지`만 짧게 적는다.
