# 2026-04-13 transcript timestamp UX A/B comparison

## 변경 요약
- media pipeline 페이지의 전사 타임스탬프 UI를 A/B 워킹트리로 비교했다.
- A안은 기존 상태판을 유지한 채 타임스탬프를 별도 카드로 덧붙이는 방식이었다.
- B안은 상태판을 운영 요약으로 줄이고, 전사 타임라인을 작업 공간처럼 우측에 배치하는 방식이었다.

## 선택 결과
- B안을 채택했다.

## 선택 이유
- 사용자가 먼저 봐야 하는 정보가 전사 타임라인이라서, 타임스탬프를 별도 보조 카드로 두는 A안보다 우측 작업 공간으로 올린 B안이 더 읽기 쉽다.
- 상태판의 내부 정보는 왼쪽 요약 카드로 줄이고, 타임라인을 오른쪽에 배치하면 시선 이동이 단순해진다.
- 요약, 검색, 숏폼이 모두 같은 시간축을 쓴다는 메시지를 B안에서 더 자연스럽게 전달할 수 있다.

## 제외한 방향
- A안처럼 기존 상태판 아래에 전사 카드만 추가하는 구성은 정보가 계속 아래로 쌓여서 핵심 흐름이 분산된다.
- 상태판 상세를 그대로 유지한 채 타임라인만 덧붙이면 화면이 여전히 길어지고, 사용자가 지금 해야 할 행동이 흐려진다.
