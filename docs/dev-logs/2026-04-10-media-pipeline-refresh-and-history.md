# 미디어 파이프라인 새로고침과 추출 이력 보강

## 왜 바꿨는가
- 최신 상태 보강 이후에도 사용자가 callback 반영을 직접 다시 확인할 수 있는 수동 새로고침 동선이 있으면 운영 확인이 더 쉬웠다.
- 최신 extraction 하나만 보여주면 실패 이력이나 직전 시도와 현재 시도를 비교하기 어려웠다.

## 무엇을 바꿨는가
- 파이프라인 상태 보드에 수동 `상태 새로고침` 버튼을 추가했다.
- 최근 extraction 이력을 최대 3건까지 노출해 생성 시각, 갱신 시각, 상태, audio URL, processing job, 오류를 함께 보이게 했다.
- 페이지에서 이미 조회 중이던 extraction 목록을 상태 보드에 전달하도록 연결했다.

## A/B 비교와 선택 이유
- A안: 기존 상태 보드 안에 수동 새로고침과 최근 이력을 같이 보강하는 방식
- B안: 별도 extraction history 패널과 훅을 새로 분리하는 방식
- 이번에는 A안을 선택했다. 파일 수와 변경량이 적고, 현재 페이지 책임 안에서 충분히 읽히며 검증도 단순했다.

## 어떻게 검증했는가
- `npm exec --workspace @myway/frontend -- tsc -p tsconfig.json --noEmit`
