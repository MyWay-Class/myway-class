# 2026-04-09 커스텀 코스와 숏폼 라이브러리 A/B 비교

## 한줄 요약
기존 `숏폼 제작`과 `내 숏폼` 화면을 확장한 A안과, `리믹스 허브` 전용 페이지를 추가한 B안을 비교했고 A안을 선택했다.

## 구현 내용
- custom course / shortform 라이브러리 조회용 frontend API를 추가했다.
- `숏폼 제작` 화면에서 개인 코스 조립, 공유, 커뮤니티 미리보기를 붙였다.
- `내 숏폼` 화면에서 owned/copied/saved 상태를 나눠 개인 코스와 숏폼 라이브러리를 정리했다.
- B안은 `리믹스 허브` 단일 페이지와 탭 구조로 같은 데이터를 더 분리된 진입점에서 보게 만들었다.

## A/B 비교
- A안은 기존 메뉴와 화면 흐름을 유지해서 사용자 학습 비용이 낮다.
- B안은 구조는 더 선명하지만 새 진입점과 탭이 늘어나 운영 화면이 더 무거워진다.
- 이슈 요구사항은 데이터 흐름 정리와 라이브러리 분리가 핵심이라, 기존 페이지를 확장하는 A안이 더 자연스럽다.

## 검증
- `npm run build:backend` 통과
- `cmd /c node_modules\\.bin\\tsc -p frontend\\tsconfig.json --noEmit` 통과
- `npm run build:frontend`는 현재 워킹트리 환경의 Vite `spawn EPERM` 문제로 추가 확인이 필요할 수 있다
