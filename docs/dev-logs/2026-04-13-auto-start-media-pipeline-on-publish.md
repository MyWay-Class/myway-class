# 2026-04-13 Auto Start Media Pipeline on Publish

## 변경 요약
- 강의 스튜디오에서 발행 준비가 끝난 뒤 미디어 파이프라인을 자동 시작하도록 연결했다.
- 발행 대상 차시에 영상 소스가 있으면 오디오 추출과 STT 처리를 바로 시도한다.
- 영상 소스가 없거나 자동 시작이 실패해도 강의 발행 자체는 막지 않도록 했다.

## 의사결정
- 선택한 안: 발행 완료 직후 `LectureStudioPage`에서 미디어 파이프라인을 바로 시작하는 최소 결합안.
- 선택 이유: 사용자가 체감하는 지점이 "발행 직후 STT가 바로 돌아가느냐"이므로, 별도 중간 화면보다 기존 발행 버튼 흐름에 붙이는 편이 자연스럽다.
- 보류한 안: 발행 전/후를 나누는 별도 A/B 비교 화면이나, 파이프라인 상태를 먼저 고른 뒤 시작하는 2단계 플로우.
- 보류 이유: 이번 이슈의 핵심은 UX 비교보다 연결 누락 해소이며, 단계가 늘어나면 발행과 전사 사이의 경로가 길어져 사용자가 더 헷갈릴 수 있다.
