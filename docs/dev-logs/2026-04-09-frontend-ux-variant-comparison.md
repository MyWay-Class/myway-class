# 2026-04-09 프론트 UX 2안 비교

## 변경 이유
- 이슈 `#46`은 단순 디자인 보정보다 범위가 크고, 레퍼런스 LMS 구조를 기준으로 프론트를 다시 잡아야 하는 작업이었다.
- `agent.md` 기준으로 비교 여지가 큰 프론트 작업이므로 워킹트리 2안 비교를 남길 필요가 있었다.

## 변경 내용
- `feat/46-frontend-ux-a`는 기존 MyWayClass 기능 패널 조합을 유지하면서 화면 톤을 정리하는 방향으로 유지했다.
- `feat/46-frontend-ux-b`는 `frontend/src/features/lms/` 아래에 로그인, 앱 셸, 권한별 메뉴, 역할별 페이지를 분리해 레퍼런스 LMS 구조에 맞게 다시 구성했다.
- B안은 `TypeScript + Tailwind` 기준으로 재구성했고, 기존 `components/dashboard`, `components/domain`, `components/ui` 레거시 묶음은 제거했다.
- B안에는 운영자 화면(`대시보드`, `사용자 관리`, `강사 관리`, `강사·수강생 배정`, `통계 현황`, `업무 자동화`)과 학습 도구 화면(`강의 목록`, `숏폼`, `커뮤니티`, `AI 챗`, `AI 강의 요약`, `시험·퀴즈 자동 생성`, `과제 검사`, `내 숏폼`)을 우선 연결했다.
- 초기 데이터 로딩은 병렬화해서 체감 반응 속도를 함께 줄였다.

## 비교 메모
- A안은 기존 기능 구성을 보존하는 데 유리했지만, 레퍼런스와 같은 `로그인 -> 앱 셸 -> 권한별 화면` 흐름을 만들기에는 구조적 한계가 있었다.
- B안은 변경 파일 수는 더 많았지만, 레퍼런스 적합성, 역할별 정보 구조, 파일 분리 적합성에서 우세했다.
- 비교 기준은 `변경량 / 파일 수 / 검증 결과 / 파일 분리 적합성`으로 봤고, 최종 선택 후보는 B안으로 정리했다.

## 검증
- `npm run build:frontend` 통과
- B안 기준 브랜치: `feat/46-frontend-ux-b`
