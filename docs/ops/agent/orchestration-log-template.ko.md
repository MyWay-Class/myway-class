# 오케스트레이션 실행 로그 템플릿 (UTF-8)

## 1) 기본 메타
- 실행 ID: `<run-YYYYMMDD-XXX>`
- 실행 일시(KST): `<YYYY-MM-DD HH:mm>`
- 매니저: `<manager name/role>`
- 대상 범위: `<기능/버그/리팩터링/문서>`
- 관련 브랜치: `<branch>`
- 관련 이슈/PR: `<#id 또는 링크>`

## 2) 매니저 의사결정
- 결정 요약: `<한 줄 요약>`
- 결정 근거:
  - `<근거 1>`
  - `<근거 2>`
- 우선순위: `<P0/P1/P2>`
- 스코프 제약:
  - `<포함>`
  - `<제외>`

## 3) 서브에이전트 논의 포인트
- 참여 서브에이전트:
  - `<name A (role)>`
  - `<name B (role)>`
- 핵심 논의:
  - `<쟁점 1>`
  - `<쟁점 2>`
- 합의 내용:
  - `<합의 1>`
  - `<합의 2>`
- 미해결/추적 항목:
  - `<todo 1>`
  - `<risk 1>`

## 4) 리뷰 체크포인트
- 코드 리뷰 체크:
  - [ ] 요구사항 충족 확인
  - [ ] 회귀 영향 점검
  - [ ] 테스트/검증 결과 첨부
- QA 체크:
  - [ ] 재현 절차 검증
  - [ ] 예외/경계값 점검
  - [ ] 문서 동기화
- 보안 체크:
  - [ ] 비밀정보 노출 여부
  - [ ] 권한/입력 검증

## 5) 병합(merge) 결과 필드
- 병합 여부: `<Merged / Not Merged>`
- 병합 일시(KST): `<YYYY-MM-DD HH:mm 또는 N/A>`
- 병합 방식: `<squash / merge commit / rebase / N/A>`
- 대상 브랜치: `<main/develop/...>`
- 최종 커밋 SHA: `<sha 또는 N/A>`
- CI 상태: `<pass/fail/pending>`
- 후속 작업:
  - `<follow-up 1>`
  - `<follow-up 2>`
