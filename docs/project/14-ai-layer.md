# AI 레이어

## 목적
기본 LMS 위에 얹는 AI 기능의 역할과 경계를 정의한다.

## 선택 이유
- AI 기능을 기본 LMS와 분리해야, AI가 없어도 LMS가 독립적으로 동작한다.
- STT, RAG, 인텐트, 요약, 숏폼은 각자 다른 문제를 풀기 때문에 한 덩어리로 두면 유지보수가 어려워진다.
- 교육 도메인에서는 정확성, 추적성, fallback이 중요해서 AI 레이어를 명시적으로 두는 편이 안전하다.
- 강의 숏폼화와 커스텀 강의 조립은 이 프로젝트의 핵심 차별점이므로, AI가 이를 안정적으로 뒷받침해야 한다.

## 범위
- STT
- 청킹
- 임베딩
- RAG
- 인텐트와 NLU
- 챗봇
- 요약
- 숏폼 생성
- 퀴즈와 플래시카드
- 커스텀 강의용 클립 선택 보조

## 입력
- 강의 텍스트
- 미디어 자산
- 사용자 질문
- 수강 맥락

## 출력
- 요약 결과
- 답변
- 숏폼 초안
- 클립 후보
- 검색 근거
- 의도 분류 결과

## 규칙
- AI 기능은 기본 LMS와 분리한다.
- 사용자 질문은 먼저 인텐트와 맥락을 해석한 뒤 처리한다.
- 근거 없는 답변보다 실패를 명시하는 편을 우선한다.
- RAG는 강의 근거를 우선 사용한다.
- 숏폼은 강의 내용에서 파생된 것만 허용한다.
- AI는 항상 원본 강의와 연결된 상태로 저장한다.
- AI는 여러 강의 영상에서 선택된 클립을 추적 가능하게 유지해야 한다.
- 실패 시에는 빈 응답보다 이유가 있는 실패 응답을 우선한다.

## 처리 순서
1. 미디어가 있으면 STT를 수행한다.
2. 텍스트를 청킹한다.
3. 청크를 임베딩한다.
4. 질의와 청크를 매칭한다.
5. 인텐트를 분류한다.
6. 답변, 요약, 숏폼, 퀴즈 중 필요한 출력을 만든다.
7. 필요한 경우 여러 영상의 클립 후보를 정리해 커스텀 강의 조립을 지원한다.

## 기준값
- STT는 미디어 입력이 있을 때만 수행한다.
- 청킹은 RAG와 요약의 품질을 해치지 않을 정도의 크기로 나눈다.
- 인텐트 confidence가 낮으면 확인 질문을 보낸다.
- 요약과 숏폼은 원문 추적이 가능해야 한다.
- 커스텀 강의용 클립은 영상 번호와 구간이 다시 추적 가능해야 한다.
- 챗봇은 강의 맥락과 일반 질문을 구분한다.

## 예외 상황
- 텍스트 전용 강의는 STT를 생략할 수 있다.
- 단순 질문은 RAG 없이 직접 답할 수 있다.

## 실패 모드
- AI 결과가 원본 강의와 연결되지 않는 경우
- 인텐트가 잘못 분류되어 엉뚱한 기능이 실행되는 경우
- 숏폼이 강의 사실과 어긋나는 경우

## 검증 기준
- 각 AI 기능의 책임이 겹치지 않는다.
- 실패 시 fallback 경로가 문서로 설명된다.

## Provider 계층
- 현재 구현은 `demo` 엔진을 기본 동작으로 유지한다.
- 향후 운영 경로는 `Ollama -> Gemini -> Cloudflare AI -> demo` 순의 fallback 계층을 기본으로 둔다.
- `STT`와 `embedding`은 `Cloudflare AI`를 우선 고려하고, 텍스트 생성 계열은 `Ollama`를 우선 고려한다.
- provider 선택과 fallback 순서는 `GET /api/v1/ai/providers`로 조회할 수 있다.

## 현재 구현
- `POST /api/v1/ai/intent`로 사용자 메시지의 의도를 분류한다.
- `POST /api/v1/ai/search`로 강의 본문, 트랜스크립트, 요약 노트에서 근거를 찾는다.
- `POST /api/v1/ai/answer`로 질문 응답과 근거 참조를 함께 돌려준다.
- `POST /api/v1/ai/summary`로 강의 요약을 생성한다.
- `POST /api/v1/ai/quiz`로 강의 기반 퀴즈를 생성한다.
- `GET /api/v1/ai/insights`로 AI 사용량과 역할별 인사이트를 조회한다.
- `GET /api/v1/ai/providers`로 provider 계층과 fallback 순서를 조회한다.
- `POST /api/v1/smart/chat`로 의도 분류와 응답 라우팅을 한 번에 처리한다.
- 공통 로직은 `packages/shared/src/ai.ts`에서 관리한다.
