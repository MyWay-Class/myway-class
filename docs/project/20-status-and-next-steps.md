# 현재 상태와 다음 단계

## 목적
지금까지 구현된 기능을 한눈에 정리하고, 등록 이슈가 모두 끝난 뒤 다음에 어떤 기능을 우선 검토할지 정리한다.

## 현재까지 완료된 기능
- LMS 코어 흐름
  - 로그인, 역할별 화면 전환, 수강 신청, 강의 완료, 자료/공지 등록
  - 학생, 강사, 운영자별 기본 대시보드
- AI 레이어
  - AI provider 카탈로그와 adapter 분리
  - 실제 엔진 연결 가이드 문서화
  - 실제 Ollama provider 연결과 fallback / timeout 정책 고정
  - AI 요약, 퀴즈, 채팅, 인사이트 흐름 정리
- STT / 미디어
  - STT provider adapter와 실제 Cloudflare 전사 경로 연결
  - 강의 전사와 요약, 타임라인형 요약 데이터 구조 정리
- 커스텀 코스와 숏폼
  - 커스텀 코스 조립과 숏폼 라이브러리 흐름
  - 공유, 담아가기, 개인 라이브러리 분류
- 코스 메타데이터 상세화
  - 썸네일 팔레트, 평점, 수강생 수, 총 러닝타임
  - 비디오 URL, transcript excerpt, keywords
  - 코스 상세 페이지 확장
- 대시보드 활동 타임라인
  - 최근 활동, 학습 통계, 역할별 인사이트 카드
  - 학생, 강사, 운영자별 요약 문구와 타임라인
- 활동 / AI 로그 누적 기록 기반
  - AI 사용 로그와 질문 로그를 append 방식으로 누적
  - 대시보드 최근 활동이 누적 로그를 다시 읽도록 정리
- 대시보드 및 상세 UX
  - 모바일에서 사이드바를 숨기고 콘텐츠 우선 레이아웃으로 정리
  - 로딩, 오프라인, 빈 상태를 공통 상태 패널로 통일
  - 코스 상세와 강의 상세에 선택 안내 카드 추가
- 운영 도구 고도화
  - 강사/운영자 화면에 검색, 필터, 정렬을 추가하고 역할별 우선순위를 다시 정리
  - AI 로그 비교 카드를 넣어 최근 집계와 이전 집계를 함께 확인
- 배포 / 인프라 문서
  - Cloudflare 배포 바인딩과 런타임 env 정리
  - 작업 규칙, 폴더/파일 분리 기준, dev log 기준 문서화

## 앞으로 해야 할 기능
- 실제 엔진 연결
  - AI provider는 Ollama 실제 호출 경로를 붙였고, 이후 Gemini / Cloudflare 확장을 검토한다.
  - STT provider는 실제 호출 경로를 붙였으니, 이후 다른 provider 확장을 검토한다.
- 데이터 지속성
  - 데모 메모리 기반 append 구조를 유지하면서, DB나 영속 저장소 이관 시점을 검토한다.
  - 활동 타임라인과 AI 로그를 저장소 추상화로 분리할지 검토한다.
- 운영 도구 고도화
  - 운영자/강사 화면의 세부 비교 차트를 더 정교하게 다듬는다.
- UX 다듬기
  - 대시보드와 상세 페이지의 CTA 위치를 더 다듬는다.
  - 필요하면 남은 페이지의 빈 상태 문구를 추가 정리한다.
- 검증 / 배포 안정화
  - backend workspace 의존성 점검을 자동화한다.
  - frontend build 환경 오류 같은 실행 이슈를 별도로 정리한다.
  - `npm run verify`로 의존성과 빌드를 한 번에 점검한다.

## 판단 기준
- 새 이슈가 생기면 기능 단위로 작게 쪼갠다.
- A/B 분기가 필요한 경우에만 워킹트리를 쓴다.
- 문서보다 구현이 먼저 필요한지, 아니면 설계 문서만으로 충분한지 먼저 판단한다.
- 등록 이슈가 끝났으므로 앞으로는 유지보수성, 운영 안정성, 실제 엔진 연결 순으로 우선순위를 잡는다.
