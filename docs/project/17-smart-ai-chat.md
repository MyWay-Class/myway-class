# 스마트 AI 통합 채팅

## 목적
강의 페이지와 코스 페이지에서 사용자의 질문을 하나의 진입점으로 받아, 의도에 따라 요약, 질문응답, 검색, 번역, 비교를 라우팅한다.

## 선택 이유
- 사용자가 기능 버튼을 외우지 않아도 된다.
- AI core의 개별 엔드포인트를 감추고, 대화형 흐름으로 보여줄 수 있다.
- 강의 단위와 코스 단위 문맥을 함께 다룰 수 있다.

## 범위
- `/api/v1/smart/chat`
- 의도 분류
- 요약 라우팅
- 질문응답 라우팅
- 검색 라우팅
- 번역 라우팅
- 비교 라우팅

## 입력
- 사용자 메시지
- 강의 식별자
- 코스 식별자
- 대화 문맥

## 출력
- 응답 본문
- 참조 근거
- 추가 질문 제안
- 라우팅 결과

## 규칙
- 스마트 채팅은 먼저 의도를 분류한 뒤 처리한다.
- 강의 식별자가 있으면 강의 근거를 우선 쓴다.
- 코스 식별자가 있으면 같은 코스의 강의 맥락을 함께 본다.
- 번역과 비교는 일반 질문과 구분해 라우팅한다.
- 실패하면 빈 응답보다 이유가 있는 응답을 돌려준다.

## 기준값
- 하나의 엔드포인트로 대화 흐름을 시작할 수 있어야 한다.
- 요약, 검색, 질문응답은 기존 AI 레이어 결과와 일관돼야 한다.
- 코스 문맥이 있으면 관련 강의 근거를 함께 보여줄 수 있어야 한다.
- 프론트 채팅 화면은 데스크톱 고정 2컬럼과 모바일 토글 구조를 따라야 한다.

## 예외 상황
- 강의나 코스가 없으면 일반 질문으로 처리할 수 있다.
- 의도가 애매하면 확인 질문을 돌려준다.

## 실패 모드
- 의도가 잘못 분류되어 엉뚱한 결과가 나오는 경우
- 강의/코스 문맥이 있는데도 일반 질문처럼 처리되는 경우

## 검증 기준
- `/api/v1/smart/chat` 하나로 AI 흐름을 시작할 수 있다.
- 강의와 코스 문맥을 함께 처리할 수 있다.
