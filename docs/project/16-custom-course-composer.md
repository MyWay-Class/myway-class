# 커스텀 강의 조립

## 목적
같은 강사와 같은 강의를 듣는 수강생이 여러 영상에서 원하는 구간을 골라 하나의 커스텀 학습 강의로 재조립할 수 있게 한다.

## 선택 이유
- 수강생은 긴 20개 영상 전체를 다시 보는 것보다 필요한 구간만 모아 복습하고 싶어한다.
- 여러 영상의 핵심 구간을 한 번에 이어 붙이면 학습 효율이 높아진다.
- 이 기능은 단순 숏폼이 아니라 강의를 개인화된 학습 자산으로 재구성하는 핵심 차별점이다.
- 같은 강의 수강생끼리만 공유하면 강의 맥락과 저작권 경계를 지킬 수 있다.
- 같은 강사, 같은 강의, 같은 수강 집단이라는 경계가 분명할수록 담아가기와 재학습의 의미가 선명해진다.

## 범위
- 영상별 구간 선택
- 여러 클립 조립
- 커스텀 강의 저장
- 같은 강의 수강생끼리의 공유
- 담아가기

## 입력
- 강사 식별자
- 강의 식별자
- 영상 목록
- 각 영상의 transcript와 timestamp
- 사용자가 고른 시작/끝 구간

## 출력
- 커스텀 강의 타임라인
- 연결된 클립 목록
- 요약 제목과 설명
- 공유 가능한 재구성 강의

## 규칙
- 커스텀 강의는 반드시 원본 강의 클립에서만 만들어야 한다.
- 한 커스텀 강의는 여러 영상의 여러 구간을 포함할 수 있다.
- 공유 범위는 같은 강사, 같은 강의, 실제 수강생으로 제한한다.
- 공유 가능 여부는 enrollment로 검증한다.
- 담아가기도 동일한 강의 수강 집단 안에서만 허용한다.
- 각 클립은 원본 영상과 시작/끝 시간을 추적 가능하게 저장한다.
- 편집 순서는 사용자가 정한 타임라인을 따른다.

## 상태
- `DRAFT`
- `COMPOSING`
- `READY`
- `SHARED`
- `COPIED`
- `ARCHIVED`

## 기준값
- 커스텀 강의는 20개 영상 중 필요한 구간만 조합할 수 있어야 한다.
- 하나의 클립은 초 단위 또는 분 단위로 자를 수 있어야 한다.
- 최종 결과는 학습 가능한 길이로 유지되어야 한다.
- 공유 피드는 같은 course_id 내에서만 노출되어야 한다.
- 커스텀 강의의 공유/담아가기는 같은 강사와 같은 강의로만 제한된다.

## 예외 상황
- 같은 강의 수강생이 아니면 공유된 커스텀 강의를 볼 수 없다.
- 수강 해제 시 비공개 또는 접근 제한 정책을 다시 적용할 수 있다.
- 원본 영상이 수정되면 클립의 추적 메타데이터를 다시 검증해야 한다.

## 실패 모드
- 강의 수강자가 아닌 사람이 커스텀 강의를 보는 경우
- 원본 클립 추적이 끊겨 출처를 알 수 없는 경우
- 서로 다른 강의나 강사의 클립이 섞이는 경우

## 검증 기준
- 사용자는 여러 영상에서 원하는 구간만 모아 하나의 커스텀 강의를 만들 수 있다.
- 같은 강의를 듣는 수강생끼리만 공유와 담아가기가 가능하다.
- 커스텀 강의는 원본 강의와 다시 연결해서 추적할 수 있다.

## 현재 구현
- `POST /api/v1/custom-courses/compose`로 커스텀 강의를 생성한다.
- `GET /api/v1/custom-courses/my`로 내 커스텀 강의 목록을 조회한다.
- `GET /api/v1/custom-courses/community`로 같은 강의 수강생의 공유본을 조회한다.
- `GET /api/v1/custom-courses/:customCourseId`로 상세와 클립을 확인한다.
- `POST /api/v1/custom-courses/:customCourseId/share`로 같은 강의 커뮤니티에 공유한다.
- `POST /api/v1/custom-courses/:customCourseId/copy`로 내 라이브러리에 담아간다.
- 공통 로직은 `packages/shared/src/custom-course.ts`에 둔다.
