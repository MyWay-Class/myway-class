# RAG 구조

## 목적
강의 내용이 어떻게 근거 있는 답변으로 바뀌는지 정의한다.

## 범위
- 청킹
- 임베딩
- 검색
- 재정렬
- 프롬프트 조립

## 입력
- 강의 텍스트
- 사용자 질문
- 선택적 강의 맥락

## 출력
- 정렬된 청크
- 근거가 있는 답변
- 검색 로그

## 규칙
- 일반 지식보다 먼저 강의 사실을 검색한다.
- 검색이 약하면 그 사실을 분명히 말한다.
- 프롬프트는 짧고 간결하게 유지한다.

## 파이프라인
1. 강의 텍스트를 정규화한다.
2. 의미 단위로 청크를 나눈다.
3. 임베딩을 만든다.
4. 청크 메타데이터를 D1에 저장한다.
5. 질의를 임베딩한다.
6. 벡터와 키워드 신호로 검색한다.
7. 재정렬하고 맥락을 조립한다.
8. 답변을 생성한다.

## 예외 상황
- 매우 짧은 강의는 전체 텍스트를 직접 사용할 수 있다.
- 복합 질문은 먼저 분해가 필요할 수 있다.

## 실패 모드
- 환각 답변
- 너무 많은 컨텍스트
- 품질 낮은 청크로 인한 노이즈 검색

## 검증 기준
- 답변을 근거 청크로 설명할 수 있다.
- fallback 동작이 결정적이다.
