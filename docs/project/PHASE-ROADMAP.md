# 단계별 로드맵

## 목적
MyWayClass를 어떤 순서로 완성할지 정리한다.

## 역할
이 문서는 구현 우선순위를 보여주는 상세 로드맵이다.
상세 기능 설명은 `12-lms-core.md`, `14-ai-layer.md`, `15-api-map.md`를 따른다.

## 단계
1. 기본 LMS 코어
2. 공통 엔티티와 API 계약
3. STT, 청킹, RAG, 인텐트, 요약
4. 숏폼 생성과 커뮤니티
5. 검증, 문서 정리, 배포 안정화

## 기준
- `Must`가 먼저다.
- `Should`는 다음 단계로 미룰 수 있다.
- `Could`는 여유가 있을 때만 한다.
- `Won't`는 현재 단계에서 하지 않는다.

## 검증 기준
- 현재 단계가 끝나면 다음 단계로 넘어갈 수 있어야 한다.
- 문서와 구현 순서가 어긋나지 않아야 한다.
