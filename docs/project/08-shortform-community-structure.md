## 숏폼과 커뮤니티 구조

## 목적
강의가 어떻게 공유 가능한 짧은 학습 자산으로 바뀌는지 설명한다.

## 선택 이유
- 수강생은 긴 강의를 다시 보는 것보다 짧게 재구성된 학습 자산을 더 자주 활용한다.
- 강의 내용을 숏폼으로 바꾸면 복습, 공유, 재생산이 쉬워진다.
- 이 프로젝트의 차별점은 서비스가 콘텐츠를 내려주는 것이 아니라, 사용자가 직접 학습 자산을 다시 만들게 하는 데 있다.

## 범위
- 숏폼 생성
- 커스텀 강의 조립
- 편집과 공개
- 공유와 반응
- 랭킹과 모더레이션

## 입력
- 강의 내용
- 사용자 스타일 선호
- 목표 길이

## 출력
- 숏폼 텍스트
- 핵심 포인트
- 플래시카드
- 커뮤니티 피드 항목

## 규칙
- 숏폼은 반드시 강의 내용에서 파생되어야 한다.
- 같은 강사와 같은 강의의 수강생끼리만 공유할 수 있다.
- 같은 강의의 수강 여부는 enrollment로 검증한다.
- 공개 항목에는 모더레이션 신호가 필요하다.
- 랭킹은 조회수보다 유용성을 더 중요하게 본다.
- 숏폼은 강의 원문으로 다시 추적 가능해야 한다.
- 공개 전에 교수 검수 또는 시스템 검증을 선택적으로 넣을 수 있다.

## 상태
- `DRAFT`
- `GENERATED`
- `REVIEWED`
- `PUBLIC`
- `ARCHIVED`

## 기준값
- 숏폼 길이는 복습 가능한 짧은 단위여야 한다.
- 핵심 포인트와 원문 출처가 함께 저장돼야 한다.
- 커스텀 강의는 여러 영상의 여러 구간을 이어 붙일 수 있어야 한다.
- 공유 피드는 같은 course_id 내에서만 노출되어야 한다.
- 공개 피드는 유용성 점수와 검수 상태를 함께 본다.
- 커뮤니티 신호는 추천과 랭킹에만 쓰고 원문 사실을 대체하지 않는다.

## 예외 상황
- 비공개 숏폼이 공개 피드로 유출되면 안 된다.
- 교수 검수 항목은 추가 랭킹 가중치를 받을 수 있다.

## 실패 모드
- 품질 낮은 중복 항목이 피드를 지배하는 경우
- 강의 내용으로 다시 추적할 수 없는 경우

## 검증 기준
- 사용자가 숏폼을 만들고 공유할 수 있다.
- 커뮤니티 신호를 수집하고 활용할 수 있다.
