## 숏폼과 커뮤니티 구조

## 목적
강의가 어떻게 공유 가능한 짧은 학습 자산으로 바뀌는지 설명한다.

## 범위
- 숏폼 생성
- 편집과 공개
- 공유와 반응
- 랭킹과 모더레이션

## 입력
- 강의 내용
- 사용자 스타일 선호
- 목표 길이

## 출력
- 숏폼 텍스트
- 핵심 포인트
- 플래시카드
- 커뮤니티 피드 항목

## 규칙
- 숏폼은 반드시 강의 내용에서 파생되어야 한다.
- 공개 항목에는 모더레이션 신호가 필요하다.
- 랭킹은 조회수보다 유용성을 더 중요하게 본다.

## 상태
- `DRAFT`
- `GENERATED`
- `REVIEWED`
- `PUBLIC`
- `ARCHIVED`

## 예외 상황
- 비공개 숏폼이 공개 피드로 유출되면 안 된다.
- 교수 검수 항목은 추가 랭킹 가중치를 받을 수 있다.

## 실패 모드
- 품질 낮은 중복 항목이 피드를 지배하는 경우
- 강의 내용으로 다시 추적할 수 없는 경우

## 검증 기준
- 사용자가 숏폼을 만들고 공유할 수 있다.
- 커뮤니티 신호를 수집하고 활용할 수 있다.
