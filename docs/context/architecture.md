# 시스템 아키텍처

## 목적
MyWayClass 전체의 시스템 경계와 데이터 흐름을 정리한다.

## 핵심 구조
- 프론트엔드: `frontend/`
- 백엔드: `backend/`
- 공통 계약: `packages/shared/`
- 공통 문서: `docs/`

## 동작 원칙
- 프론트는 백엔드 API만 호출한다.
- 백엔드는 D1, R2, AI 작업을 오케스트레이션한다.
- 대용량 파일과 미디어는 R2에 둔다.
- 메타데이터와 검색용 정보는 D1에 둔다.

## 데이터 흐름
1. 사용자가 강의나 콘텐츠를 등록한다.
2. 백엔드가 입력을 검증하고 저장한다.
3. 필요 시 청킹, 임베딩, STT, 요약, 숏폼 생성이 이어진다.
4. 결과는 공유 가능한 형태로 반환된다.

## 예외 상황
- 데모 전용 흐름은 문서에 별도로 표시한다.
- 무료 AI가 실패하면 fallback으로 이어진다.
